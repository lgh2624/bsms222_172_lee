Data visualization principles

11 Data visualization principles

Here, we aim to provide some general principles we can use as a guide for effective data visualization. We compare and contrast plots that follow these principles to those that don’t. The principles are mostly based on research related to how humans detect patterns and make visual comparisons. When deciding on a visualization approach, it is also important to keep our goal in mind.

We will use:

library(tidyverse)
library(dslabs)

11.1 Encoding data using visual cues

We start by describing some principles for encoding data. There are several approaches at our disposal including position, aligned lengths, angles, area, brightness, and color hue. A widely used graphical representation of percentages is the pie chart. The donut chart is an example of a plot that uses only area. In this case, simply showing the numbers is clearer. The barplot uses this approach by using bars of length proportional to the quantities of interest. In general, when displaying quantities, position and length are preferred over angles and/or area.

11.2 Know when to include 0

When using barplots, it is misinformative not to start the bars at 0. By avoiding 0, relatively small differences can be made to look much bigger than they actually are. When using position rather than length, it is then not necessary to include 0. Note that in the plot on the left, which includes 0, the space between 0 and 43 adds no information and makes it harder to compare the between and within group variability.

11.3 Do not distort quantities

If you plot a data by circle with radius in proportion to that. It will distort your data. Not surprisingly, ggplot2 defaults to using area rather than radius.

11.4 Order categories by a meaningful value

When one of the axes is used to show categories, as is done in barplots, the default ggplot2 behavior is to order the categories alphabetically. Instead, we should order by a meaningful quantity. We previously learned how to use the reorder function, which helps us achieve this goal.

11.5 Show the data

We have focused on displaying single quantities across categories. We now shift our attention to displaying data, with a focus on comparing groups. This time let’s assume that we are interested in the difference in heights between males and females. This simple ggplot2 code already generates a more informative plot than the barplot by simply showing all the data points:

data(heights)

heights %>% 
  ggplot(aes(sex, height)) + 
  geom_point()

This plot has limitations as well, since we can’t really see all the 238 and 812 points plotted for females and males respectively, and many points are plotted on top of each other. The first is to add jitter. This adds a small random shift to each point. And the second improvement comes from using alpha blending. That makes the dot transparent.

heights %>% 
  ggplot(aes(sex, height)) +
  geom_jitter(width = 0.1, alpha = 0.2)

11.6 Ease comparisons

11.6.1 Use common axes

Since there are so many points, it is more effective to show distributions rather than individual points.

heights %>% group_by(sex) %>%
  ggplot(aes(height, y= ..density..)) +
  geom_histogram(binwidth = 1, color = 'black') +
  facet_grid(.~sex)

We have to look carefully to notice that the x-axis has a higher range of values in the male histogram. Keep the axes the same when comparing data across to plots.

heights %>% group_by(sex) %>%
  ggplot(aes(height, y= ..density..)) +
  geom_histogram(binwidth = 1, color = 'black') +
  xlim(50, 84) +
  facet_grid(.~sex)

11.6.2 Align plots vertically to see horizontal changes and horizontally to see vertical changes

Aligning the plots vertically helps us see this change when the axes are fixed:

heights %>% 
  ggplot(aes(height, ..density..)) +
  geom_histogram(binwidth = 1, color="black") +
  facet_grid(sex~.)

If we want the more compact summary provided by boxplots, we then align them horizontally

 heights %>% 
  ggplot(aes(sex, height)) + 
  geom_boxplot(coef=3) + 
  geom_jitter(width = 0.1, alpha = 0.2) +
  ylab("Height in inches")

11.6.3 Consider transformations

The combination of an incorrectly chosen barplot and a failure to use a log transformation when one is merited can be particularly distorting. For example, the average of the population by each continent. If we use log scale, it is likely that we may distort the data. So, we must carefully deal with it.

11.6.4 Visual cues to be compared should be adjacent

If we compare multiple plots, the order of any labels must be identical.

11.8 Plots for two variables

In general, you should use scatterplots to visualize the relationship between two variables. However, there are some exceptions and we describe two alternative plots here: the slope chart and the Bland-Altman plot.

11.8.1 Slope charts

One exception where another type of plot may be more informative is when you are comparing variables of the same type. There is no geometry for slope charts in ggplot2, but we can construct one using geom_line.

# select regions/countries we want
west <- c("Western Europe","Northern Europe","Southern Europe",
          "Northern America","Australia and New Zealand")

# get data of life expectancy of selected countries of two year
dat <- gapminder %>% 
  filter(year%in% c(2010, 2015) & region %in% west & 
           !is.na(life_expectancy) & population > 10^7) 


dat %>%
  # set location of each point
  mutate(location = ifelse(year == 2010, 1, 2), 
         location = ifelse(year == 2015 & 
                             country %in% c("United Kingdom","Portugal"),
                           location+0.22, location),
         # 
         hjust = ifelse(year == 2010, 1, 0)) %>%
  mutate(year = as.factor(year)) %>%
  ggplot(aes(year, life_expectancy, group = country)) +
  # make lines connecting same country
  geom_line(aes(color = country), show.legend = FALSE) +
  # show texts
  geom_text(aes(x = location, label = country, hjust = hjust), 
            show.legend = FALSE) +
  xlab("") + 
  ylab("Life Expectancy")

11.8.2 Bland-Altman plot

The Bland-Altman plot, also known as the Tukey mean-difference plot and the MA-plot, shows the difference versus the average:

library(ggrepel)

dat %>% 
  # make colunm of data we want
  mutate(year = paste0("life_expectancy_", year)) %>%
  select(country, year, life_expectancy) %>%
  spread(year, life_expectancy) %>% 
  # calculate avg. and dif.
  mutate(average = (life_expectancy_2015 + life_expectancy_2010)/2,
         difference = life_expectancy_2015 - life_expectancy_2010) %>%
  
  # plotting
  ggplot(aes(average, difference, label = country)) + 
  geom_point() +
  
  # name the point and axis
  geom_text_repel() +
  geom_abline(lty = 2) +
  xlab("Average of 2010 and 2015") + 
  ylab("Difference between 2015 and 2010")

11.10 Avoid pseudo-three-dimensional plots

Humans are not good at seeing in three dimensions and our limitation is even worse with regard to pseudo-three-dimensions. This only adds confusion and makes it harder to relay your message.

11.11 Avoid too many significant digits

By default, statistical software like R returns many significant digits. That many digits often adds no information and the added the visual clutter can make it hard for the viewer to understand the message.

11.12 Know your audience

Graphs can be used for 1) our own exploratory data analysis, 2) to convey a message to experts, or 3) to help tell a story to a general audience. Make sure that the intended audience understands each element of the plot.

11.14 Case study: impact of vaccines on battling infectious diseases

Vaccines have helped save millions of lives. In the 19th century, before herd immunization was achieved through vaccination programs, deaths from infectious diseases, such as smallpox and polio, were common. However, today vaccination programs have become somewhat controversial despite all the scientific evidence for their importance. The data used for these plots were collected, organized and distributed by the Tycho Project. They include weekly reported counts for seven diseases from 1928 to 2011, from all fifty states. We include the yearly totals in the dslabs package:

library(tidyverse)
library(dslabs)
data(us_contagious_diseases)
str(us_contagious_diseases)

We create a temporary object dat:

# Select Measles only
the_disease <- "Measles"

# Calculate incident rate per 100000
dat <- us_contagious_diseases %>%
  filter(!state%in%c("Hawaii","Alaska") & disease == the_disease) %>%
  mutate(rate = count / population * 10000 * 52 / weeks_reporting) %>% 
  mutate(state = reorder(state, rate))

We can now easily plot disease rates per year. Here are the measles data from California:

dat %>%
  # select data of California
  filter(state == "California" & !is.na(rate)) %>%
  ggplot(aes(year, rate)) +
  geom_line() + 
  ylab("Cases per 10,000")  + 
  # insert the time when vaccine was introduced
  geom_vline(xintercept=1963, col = "blue")

Now can we show data for all states in one plot? We have three variables to show: year, state and rate. We use the geometry geom_tile to tile the region with colors representing disease rates.

dat %>% ggplot(aes(year, state, fill = rate)) +
  geom_tile(color = "grey50") +
  scale_x_continuous(expand=c(0,0)) +
  scale_fill_gradientn(colors = RColorBrewer::brewer.pal(9, "Reds"), trans = "sqrt") +
  geom_vline(xintercept=1963, col = "blue") +
  # make more tidy
  theme_minimal() +  theme(panel.grid = element_blank()) +
  ggtitle(the_disease) + 
  ylab("") + 
  xlab("")

This plot makes a very striking argument for the contribution of vaccines. However, one limitation of this plot is that it uses color to represent quantity. If we are willing to lose state information, we can make a version of the plot that shows the values with position.

avg <- us_contagious_diseases %>%
  filter(disease==the_disease) %>% group_by(year) %>%

Now to make the plot we simply use the geom_line geometry:

dat %>%
